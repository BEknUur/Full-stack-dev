# Облачные провайдеры: исследование и выбор наилучшего решения

## 1. AWS (Amazon Web Services)
### Описание:
AWS — один из крупнейших облачных провайдеров, предлагающий широкий спектр услуг, от виртуальных машин до хранения данных и инструментов машинного обучения.

### Типы серверов:
- **Стандартные** (General Purpose): Для общего использования (t3.micro, t3.medium).
- **Высокопроизводительные** (Compute Optimized): Для вычислительных задач (c6g, c5).
- **Для хранения данных** (Storage Optimized): Для больших баз данных и систем хранения (i3, d2).

### Операционные системы:
- Amazon Linux, Ubuntu, Red Hat, SUSE, Windows Server и другие.

### Масштабирование:
- Автоматическое масштабирование (Auto Scaling) и балансировка нагрузки (Elastic Load Balancing).

### Пример сервера:
- **t3.micro** (2 vCPU, 1 ГБ RAM, SSD 30 ГБ)
- Стоимость: $8–10 в месяц (с учётом использования в течение 750 часов в месяц).

### Причины выбора:
Идеально подходит для небольших веб-приложений и стартапов благодаря бесплатному уровню и доступной стоимости. Высокая надёжность и возможность масштабирования при росте нагрузки.

---

## 2. Google Cloud (GCP)
### Описание:
Google Cloud Platform предлагает мощные инструменты для разработчиков и корпоративных клиентов, включая поддержку контейнеров и Kubernetes.

### Типы серверов:
- **Стандартные** (e2, n2): Универсальные машины для большинства приложений.
- **Высокопроизводительные** (c2): Для задач с высокой вычислительной нагрузкой.
- **Для памяти** (m1, m2): Для приложений, требующих много памяти.

### Операционные системы:
- Ubuntu, CentOS, Debian, Windows Server и другие.

### Масштабирование:
- Поддержка автоматического масштабирования и использования Kubernetes Engine для контейнеров.

### Пример сервера:
- **e2-micro** (2 vCPU, 1 ГБ RAM, 30 ГБ SSD)
- Стоимость: $6-7 в месяц (при использовании бесплатного уровня).

### Причины выбора:
Отлично подходит для стартапов и приложений на основе контейнеров благодаря бесплатному уровню и интеграции с Kubernetes. Подходит для проектов с интенсивным использованием данных.

---

## 3. Yandex Cloud
### Описание:
Yandex Cloud — российский облачный провайдер с широким набором услуг, включая виртуальные машины, базы данных и инструменты для машинного обучения.

### Типы серверов:
- **Стандартные** (standard-v1): Для общего использования.
- **Высокопроизводительные** (compute-v1): Для ресурсоёмких задач.
- **Оптимизированные для хранения** (storage-v1): Для интенсивного ввода/вывода данных.

### Операционные системы:
- Ubuntu, CentOS, Debian, Windows Server.

### Масштабирование:
- Поддержка автоматического масштабирования и балансировки нагрузки.

### Пример сервера:
- **standard-v1** (2 vCPU, 2 ГБ RAM, 40 ГБ SSD)
- Стоимость: около 700–800 рублей в месяц (~$10).

### Причины выбора:
Подходит для развертывания локальных веб-приложений с учётом специфики рынка. Преимущество для российских пользователей благодаря локальной поддержке и меньшим задержкам.

---

## Выбор наилучшего решения

### Выбранный сервер: AWS t3.micro
**Причины выбора**:
- Доступен бесплатный уровень, что экономит бюджет для небольших проектов.
- Возможность масштабирования при увеличении нагрузки.
- Надёжная поддержка и стабильная работа для развертывания веб-приложений и API.

---

## Рекомендации для студентов

1. **Для небольших проектов с ограниченным бюджетом**:
   - Используйте **AWS (t3.micro)** или **Google Cloud (e2-micro)** для бесплатного уровня.
   - Эти варианты обеспечивают базовые вычислительные мощности и надёжность.

2. **Для проектов с большими нагрузками**:
   - Рассмотрите **Google Cloud (c2)** или **AWS (c5)** для высокопроизводительных приложений.
   - Обеспечьте балансировку нагрузки и масштабирование для устойчивости.

3. **Для локальных проектов в СНГ**:
   - Используйте **Yandex Cloud**, если вам важна локальная поддержка и низкие задержки.
   - Подходит для развертывания веб-сайтов и API для российских пользователей.
